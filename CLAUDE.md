
# network-study 프로젝트

- 네트워크의 세부 동작에 대해 이해하기 위한 가시화 학습 도구를 작성합니다.
- 이 프로젝트의 목적은 만든 본인 및 사이트 이용자가 네트워크의 각 동작에 대한 심도있는 이해를 돕기 위해 웹으로 가시화 하는 것을 목표로 합니다.
- react.js + typescript + pnpm 등을 이용한 static page를 github-page로 서빙하는 것을 목표로 합니다.
- d3, webgl, drawio 등 다채로운 도구를 활용합니다.

## Rule

- 필요하다면 CLAUDE는 skill을 만들고 이를 활용할 수 있습니다.
- 주제는 CLAUDE와 개발자가 함께 토론하여 결정합니다.
- 웹에 보여줄 목차와 내용 개요를 docs/PAGES.md 에 정리하고, 변경 시 항상 업데이트 합니다.
- 진행사항은 docs/PROCESS.md에 정리하고, 변경 시 항상 업데이트 합니다.
- UI 스타일에 대한 사항은 docs/STYLE.md에 정리하고, UI 작업 시 항상 참조합니다. 변경 시 항상 업데이트 합니다.
- **작업 완료 후 반드시 docs/PROCESS.md를 업데이트합니다.** 스프린트 또는 단위 작업이 끝날 때마다 완료 항목을 기록하고, 날짜를 최신화합니다.

## 네트워크 학습 목차

시스템 / 리눅스 / 프록시 / 패킷 처리 / 실제 망 구성 관점에서  
네트워크를 이해하기 위한 학습 목차입니다.

이 문서는 단순한 프로토콜 암기보다,  
**실제 네트워크가 어떻게 연결되고, 어떤 장비가 어떤 위치에서 어떤 역할을 하는지**를 함께 이해하는 것을 목표로 합니다.

## 1. 네트워크의 기초와 전체 구조

### 학습 목표

네트워크가 왜 계층적으로 설계되었는지 이해하고, 데이터가 어떤 단위를 거쳐 전달되는지 큰 그림을 잡습니다.

### 주요 주제

- 네트워크란 무엇인가
- LAN, WAN, Internet의 차이
- 패킷 교환 방식
- 프로토콜의 의미와 역할
- 계층형 설계의 필요성
- OSI 7계층과 TCP/IP 모델 비교
- 캡슐화 / 역캡슐화
- 프레임 / 패킷 / 세그먼트 / 스트림의 차이
- MTU의 의미

### 핵심 질문

- 왜 네트워크는 계층으로 나누어 설계되었는가?
- Ethernet 프레임과 IP 패킷, TCP 세그먼트는 어떻게 연결되는가?
- MTU는 왜 중요한가?

## 2. 실제 네트워크 망 구성의 기초

### 학습 목표

현실의 네트워크가 어떤 장비와 구간으로 구성되는지 이해하고,  
“인터넷 ↔ 라우터 ↔ 스위치 ↔ 서버/클라이언트” 구조를 머릿속에 그릴 수 있도록 합니다.

### 주요 주제

- 가정용 네트워크와 기업 네트워크의 차이
- 인터넷 회선, ISP, 공인 IP, 사설 IP
- 라우터 / L3 스위치 / L2 스위치의 역할 차이
- 액세스 스위치, 코어 스위치, 배포 스위치 개념
- 서버망, 사용자망, 관리망 분리
- DMZ의 개념과 목적
- 내부망 / 외부망 / 인터넷 구간의 차이
- 방화벽이 들어가는 위치
- 프록시 / 보안장비 / 로드밸런서의 일반적 배치 위치
- 이중화된 네트워크 구성의 기초

### 핵심 질문

- 집 네트워크와 회사 네트워크는 구조적으로 무엇이 다른가?
- 왜 망을 사용자망, 서버망, 관리망으로 나누는가?
- 보안장비는 보통 어느 위치에 배치되는가?

## 3. 물리 계층과 링크 계층

### 학습 목표

실제 NIC, MAC 주소, 스위치, VLAN 등 L2 중심 개념을 익히고, 같은 네트워크 안에서 프레임이 어떻게 전달되는지 이해합니다.

### 주요 주제

- Ethernet의 기본 구조
- NIC(Network Interface Card)의 역할
- MAC 주소의 의미
- 스위치의 동작 원리
- MAC address table
- ARP와 Gratuitous ARP
- 브로드캐스트 / 멀티캐스트 / 유니캐스트
- VLAN과 802.1Q
- 액세스 포트와 트렁크 포트
- bonding / teaming / LACP 기초
- speed / duplex / link up-down 상태 이해

### 핵심 질문

- 스위치는 어떤 기준으로 프레임을 전달하는가?
- ARP는 왜 필요한가?
- VLAN은 어떤 문제를 해결하기 위해 쓰이는가?

## 7. 실제 서비스 망 설계와 트래픽 흐름

### 학습 목표

현실의 서비스 환경에서 사용자의 요청이 어떤 장비를 통과해 서버까지 도달하는지 이해합니다.

### 주요 주제

- 클라이언트 → 스위치 → 라우터 → 방화벽 → LB → 서버 기본 흐름
- 사내 사용자 인터넷 접속망의 일반 구조
- 서버팜 / 웹서버 / WAS / DB 분리 구조
- DMZ에 웹 서버를 두는 이유
- 인터넷 공개 서비스망과 내부 업무망의 분리
- 동서(East-West) 트래픽과 남북(North-South) 트래픽
- NAT가 적용되는 위치
- 프록시 기반 트래픽 우회 구조
- 보안 검사 장비(방화벽, IPS, SWG, Proxy)의 삽입 위치
- 고가용성 구성을 위한 Active-Standby / Active-Active 기초

### 핵심 질문

- 사용자의 웹 요청은 실제 망에서 어떤 장비들을 통과하는가?
- 서버는 왜 한 구역에 몰아넣지 않고 역할별로 나누는가?
- 인터넷 경계망과 내부 서비스망은 왜 분리하는가?

## 10. 성능과 트래픽 제어

## 실습 예제 섹션에 넣기 좋은 주제

- `ip addr`, `ip route`, `ip rule` 실습
- `arp -n` 또는 `ip neigh` 확인
- `tcpdump`로 TCP handshake 캡처
- `ss -tnp`로 소켓 상태 확인
- `ethtool -k`, `ethtool -S` 확인
- `iptables` / `nft` 규칙 관찰
- `traceroute`, `mtr` 사용
- `ping -M do`로 MTU 관련 실험
- 네임스페이스 + veth로 가상 네트워크 구성
- 간단한 NAT / 포트포워딩 실험
- 프록시 또는 로드밸런서 앞뒤 흐름 관찰
- VLAN 분리 실험
- 이중 NIC / bonding 구성 실험

## 이 목차의 특징

이 목차는 단순 입문용 네트워크 이론보다 다음 관점을 더 강조합니다.

- 실제 망이 어떻게 배치되는가
- 장비가 어떤 위치에서 어떤 역할을 하는가
- 리눅스와 네트워크가 어떻게 만나는가
- 프록시 / 방화벽 / NAT / 라우팅 실무
- 성능 병목과 고속 처리 구조
- 장애 분석과 운영 관점

즉, 단순히 “OSI 7계층을 외우는 네트워크”가 아니라  
“현실의 네트워크 망이 어떻게 구성되고, 그 안에서 패킷이 어떻게 흐르는지”를 배우기 위한 구조입니다.

## 한 줄 요약

이 문서는 네트워크를  
**기초 구조 → 실제 망 구성 → 링크 계층 → IP/라우팅 → TCP/IP → 응용 프로토콜 → 서비스 흐름 → 리눅스 스택 → 방화벽/NAT/프록시 → 성능 → 장애 분석 → 현대 아키텍처**  
순서로 학습하기 위한 전체 목차입니다.
